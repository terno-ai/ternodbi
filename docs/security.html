Security & Tokens

Security is the foundation of TernoDBI. We use a multi-layered approach to ensure your data stays safe while being accessible to AI.

1. SQLShield: The Query Firewall

SQLShield is our proprietary SQL validation engine that parses SQL into an AST to check for forbidden node types.

  • ReadOnly Enforcement: Rejects all mutations (INSERT, UPDATE, DELETE, etc.).
  • Limit Enforcement: Automatically appends LIMIT to prevent heavy queries.

2. Service Token Authentication

TernoDBI uses scoped API Key authentication designed for agents.

Token Types

  • Query Token: Read-only access to specific datasources.
  • Admin Token: Full access to the Admin API.

Issuing a Token

ternodbi manage issue_token --name "Agent 1" --type query

3. Best Practices

  • Always use read-only database users for the connection string.
  • Rotate tokens regularly.
  • Use scoped tokens whenever possible.
