Setup & Installation

Welcome to TernoDBI. This guide will help you get started with installing and configuring TernoDBI to bridge your databases with AI agents.

Installation

pip install terno-dbi

Configuration

TernoDBI uses environment variables for configuration. Create a .env file in your project root or export variables directly.

Core Settings

  • DATABASE_ENGINE: Supported values: POSTGRESQL, MYSQL, SNOWFLAKE, BIGQUERY, DATABRICKS, ORACLE.
  • SECRET_KEY: A secure random string for encryption.
  • DEBUG: Set to True for development, False for production.

Database Specific

Depending on your DATABASE_ENGINE, set the appropriate credentials (e.g., POSTGRES_DB, POSTGRES_USER, etc.).

Running the Server

Start the server with a single command:

ternodbi start

This command handles migrations, default user creation, and starts the API host.
