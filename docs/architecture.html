TernoDBI Architecture

TernoDBI acts as an intelligence and security layer between AI agents and your databases. It translates high-level requests into secure, optimized SQL.

1. Model Context Protocol (MCP) Interface

TernoDBI natively supports MCP, providing standardized tools for agents like Claude and Cursor.

  • Query Server: Provides read-only access to data.
  • Admin Server: Allows agents to explore and enrich schema metadata.

2. SQLShield Engine

The core security component that inspects every query before execution.

  • Validation: Rejects mutations like INSERT, DROP, or ALTER.
  • Transformation: Can rewrite queries to apply RLS or force LIMITs.
  • Dialect Translation: Handles differences between Postgres, MySQL, Snowflake, etc.

3. Unified Connector System

A factory-based abstraction over SQLAlchemy and native drivers, providing a single interface for all supported databases.

4. Service Token Authentication

Custom authentication designed for agents, with scoped access and expiration auditing.
